# TAVILY Skill - Preflight Safety Checklist

ใช้เช็กลิสต์นี้ก่อนเปิดใช้งานจริงทุกครั้ง

## 1) Secrets / Keys
- [ ] เก็บ `TAVILY_API_KEY` ในไฟล์ `.env` เท่านั้น
- [ ] ห้าม hardcode key ในโค้ด
- [ ] มี `.env` อยู่ใน `.gitignore`
- [ ] ตรวจว่าก่อน push ไม่มี key หลุดใน commit (`git diff --cached`)

## 2) Source Trust
- [ ] กำหนดโดเมนหลักที่เชื่อถือได้ (เช่น bbc.com, reuters.com, thairath.co.th)
- [ ] ถ้าเป็นข่าวสำคัญ ให้มีอย่างน้อย 2 แหล่ง
- [ ] เนื้อหาเว็บทั้งหมดถือเป็น untrusted input

## 3) Output Rules
- [ ] คำตอบต้องมี URL อ้างอิงเสมอ
- [ ] แยก “ข้อเท็จจริง” กับ “ข้อสรุป/ความเห็น”
- [ ] ถ้าข้อมูลไม่พอ ให้ตอบว่า “ยังไม่พอ” ไม่เดา

## 4) Cost / Stability
- [ ] ตั้ง `max_results` พอเหมาะ (แนะนำเริ่ม 5-10)
- [ ] ตั้ง timeout (เช่น 20-30s)
- [ ] จำกัดจำนวนคำขอต่อรอบ เพื่อคุมค่าใช้จ่าย

## 5) Local Machine Fit (เครื่อง RAM 4GB)
- [ ] เปิดเฉพาะงานที่ต้องใช้จริง (ปิดแท็บ Chrome ที่ไม่จำเป็น)
- [ ] เริ่มจาก query เล็กก่อน แล้วค่อยเพิ่มความลึก
- [ ] หลีกเลี่ยงการ crawl ใหญ่พร้อมกันหลายโดเมน

## 6) Final Sanity Check
- [ ] ตรวจว่าไม่มีคำสั่งแปลกจากหน้าเว็บที่ถูก “ทำตาม” โดยอัตโนมัติ
- [ ] สรุปสุดท้ายอ่านรู้เรื่อง กระชับ และตรวจสอบได้
