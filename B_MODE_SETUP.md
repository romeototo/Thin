# B_MODE_SETUP.md

โหมด B = คุยกับ `main` ตัวเดียว แล้วให้ `main` กระจายงานไปเอเจนต์เฉพาะทาง (trend/research/content/qa)

## เป้าหมาย
- ผู้ใช้คุยจุดเดียว
- งานหลังบ้านแยกตามบทบาท
- ลดความซับซ้อนการดูแลหลายบอท

## คำสั่งใช้งาน (สำหรับ main)
- ถ้าผู้ใช้ขอหาเทรนด์ → ส่งงานให้ `trend`
- ถ้าผู้ใช้ขอวิเคราะห์เชิงลึก → ส่งงานให้ `research`
- ถ้าผู้ใช้ขอเขียนโพสต์/แคปชัน → ส่งงานให้ `content`
- ถ้าผู้ใช้ขอตรวจคุณภาพ/ปรับภาษา → ส่งงานให้ `qa`

## Routing Rule (เชิงนโยบาย)
1) main รับคำสั่ง
2) จำแนก intent
3) delegate ให้เอเจนต์เฉพาะทาง
4) รวมผลกลับมาที่ main
5) ส่งคำตอบสุดท้ายให้ผู้ใช้

## หมายเหตุ
- ถ้า gateway pairing/เครื่องมือ session ใช้ไม่ได้ชั่วคราว ให้ main ทำงานแทนแบบ direct แล้วแจ้งสถานะ
- เมื่อระบบพร้อม ให้กลับไป delegate ตาม workflow เดิมทันที
